## 🔧 环境准备

### 1. 检查服务器环境

```bash
# 检查Python版本（建议3.7+）
python --version
# 或
python3 --version

# 检查GPU
nvidia-smi

# 检查CUDA版本
nvcc --version
```

### 2. 创建虚拟环境（推荐）

#### 使用conda（推荐）
```bash
# 创建conda环境
conda create -n nice-eeg python=3.8
conda activate nice-eeg

# 安装PyTorch（根据CUDA版本选择）
# CUDA 11.8
conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia

# CUDA 10.2
conda install pytorch torchvision torchaudio cudatoolkit=10.2 -c pytorch
```

#### 使用venv
```bash
python3 -m venv nice-eeg-env
source nice-eeg-env/bin/activate  # Linux
```

### 3. 安装依赖包

```bash
# 基础依赖
pip install numpy pandas matplotlib scikit-learn

# PyTorch相关（如果conda没安装）
pip install torch torchvision torchaudio

# 其他依赖
pip install einops transformers pillow

# 如果需要图注意力模块
pip install torch-geometric

# 如果需要EEG处理
pip install mne scipy
```

### 4. 验证安装

```bash
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

## ⚙️ 路径配置

### 1. 修改 `nice_stand.py`

需要修改以下路径（根据你的服务器实际情况）：

```python
# 第26行：GPU设置
gpus = [0]  # 改为你的GPU编号，可以用 nvidia-smi 查看
os.environ['CUDA_DEVICE_ORDER'] = 'PCI_BUS_ID'
os.environ["CUDA_VISIBLE_DEVICES"] = ','.join(map(str, gpus))

# 第29行：结果保存路径
result_path = '/home/user/results/NICE/'  # 修改为你的路径

# 第163行：EEG数据路径
self.eeg_data_path = '/home/user/Data/Things-EEG2/Preprocessed_data_250Hz/'

# 第164-165行：图像特征路径（相对路径，一般不需要改）
self.img_data_path = './dnn_feature/'
self.test_center_path = './dnn_feature/'

# 第340-342行：模型保存路径
# 确保 ./model/ 目录存在
```

**快速修改脚本**：
```bash
# 创建配置修改脚本
cat > modify_paths.sh << 'EOF'
#!/bin/bash
# 设置你的路径
PROJECT_DIR="/home/user/NICE-EEG-main"
DATA_DIR="/home/user/Data/Things-EEG2"
RESULTS_DIR="/home/user/results/NICE"
GPU_ID=0

# 修改nice_stand.py
sed -i "s|gpus = \[6\]|gpus = [$GPU_ID]|g" $PROJECT_DIR/nice_stand.py
sed -i "s|result_path = '/home/NICE/results/'|result_path = '$RESULTS_DIR/'|g" $PROJECT_DIR/nice_stand.py
sed -i "s|self.eeg_data_path = '/home/Data/Things-EEG2/Preprocessed_data_250Hz/'|self.eeg_data_path = '$DATA_DIR/Preprocessed_data_250Hz/'|g" $PROJECT_DIR/nice_stand.py

echo "路径配置完成！"
EOF

chmod +x modify_paths.sh
./modify_paths.sh
```

## 🔄 后台运行

### 方法1：使用nohup（简单）

```bash
cd /home/user/NICE-EEG-main

# 运行并将输出保存到日志
nohup python nice_stand.py \
    --dnn clip \
    --epoch 200 \
    --num_sub 1 \
    --batch-size 1000 \
    --seed 2023 \
    > training.log 2>&1 &

# 查看进程
ps aux | grep nice_stand

# 查看日志
tail -f training.log

# 停止任务（找到PID后）
kill <PID>
```

### 方法2：使用nohup + 重定向到文件

```bash
nohup python nice_stand.py \
    --dnn clip \
    --epoch 200 \
    --num_sub 1 \
    --batch-size 1000 \
    > output.log 2> error.log &
```

### 方法3：创建运行脚本

创建 `run_training.sh`：
```bash
#!/bin/bash

# 设置环境变量
export CUDA_VISIBLE_DEVICES=0
export PYTHONPATH=/home/user/NICE-EEG-main:$PYTHONPATH

# 激活conda环境（如果使用）
# source ~/anaconda3/etc/profile.d/conda.sh
# conda activate nice-eeg

# 进入项目目录
cd /home/user/NICE-EEG-main

# 运行训练
python nice_stand.py \
    --dnn clip \
    --epoch 200 \
    --num_sub 1 \
    --batch-size 1000 \
    --seed 2023

echo "Training completed at $(date)"
```

```bash
# 赋予执行权限
chmod +x run_training.sh

# 后台运行
nohup ./run_training.sh > training.log 2>&1 &
```

## 🔍 常见问题

### 1. GPU内存不足

**问题**：`RuntimeError: CUDA out of memory`

**解决方案**：
```bash
# 减小batch size
python nice_stand.py --batch-size 500  # 从1000减到500

# 或使用更少的GPU
# 修改 nice_stand.py 中的 gpus = [0]  # 只用一张卡
```

### 2. 路径不存在错误

**问题**：`FileNotFoundError: [Errno 2] No such file or directory`

**解决方案**：
```bash
# 检查路径是否存在
ls -la /home/user/Data/Things-EEG2/Preprocessed_data_250Hz/

# 检查相对路径
cd /home/user/NICE-EEG-main
ls -la ./dnn_feature/

# 确保所有目录都存在
mkdir -p model
mkdir -p /home/user/results/NICE
```

### 3. 模块导入错误

**问题**：`ModuleNotFoundError: No module named 'xxx'`

**解决方案**：
```bash
# 确保在正确的环境中
conda activate nice-eeg  # 或 source venv/bin/activate

# 安装缺失的包
pip install <module_name>

# 检查Python路径
python -c "import sys; print(sys.path)"
```

### 4. CUDA版本不匹配

**问题**：`CUDA runtime version mismatch`

**解决方案**：
```bash
# 检查CUDA版本
nvcc --version
nvidia-smi

# 重新安装匹配的PyTorch版本
# 访问 https://pytorch.org/ 获取正确的安装命令
```

### 5. 权限问题

**问题**：`Permission denied`

**解决方案**：
```bash
# 检查文件权限
ls -la

# 修改权限
chmod +x run_training.sh
chmod -R 755 /home/user/NICE-EEG-main
```

### 6. 连接断开后任务停止

**问题**：SSH断开后训练停止

**解决方案**：
- 使用 `screen` 或 `tmux`（推荐）
- 使用 `nohup`
- 使用 `systemd` 服务（高级）

```bash
# 1. SSH连接到服务器

# 6. 分离会话（Ctrl+A, D）

# 7. 重新连接查看进度
screen -r nice-sub1
```

### 示例2：多被试批量训练

创建 `batch_train.sh`：
```bash
#!/bin/bash

cd /home/user/NICE-EEG-main

for sub in {1..10}; do
    echo "Training subject $sub..."
    python nice_stand.py \
        --dnn clip \
        --epoch 200 \
        --num_sub 1 \
        --batch-size 1000 \
        --seed 2023
    
    # 等待GPU释放
    sleep 60
done
```

```bash
chmod +x batch_train.sh
nohup ./batch_train.sh > batch_training.log 2>&1 &
```

### 示例3：使用SLURM（集群环境）

创建 `train.slurm`：
```bash
#!/bin/bash
#SBATCH --job-name=nice-eeg
#SBATCH --partition=gpu
#SBATCH --gres=gpu:1
#SBATCH --cpus-per-task=4
#SBATCH --mem=16G
#SBATCH --time=48:00:00
#SBATCH --output=train_%j.out
#SBATCH --error=train_%j.err

# 加载模块（根据服务器配置）

cd /home/user/NICE-EEG-main
python nice_stand.py \
    --dnn clip \
    --epoch 200 \
    --num_sub 1 \
    --batch-size 1000
```

```bash
sbatch train.slurm
```
