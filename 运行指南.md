# NICE-EEG 项目运行指南

## 项目简介
NICE (Natural Image Contrastive EEG) 是一个基于对比学习的自监督框架，用于从EEG信号中解码自然图像，实现物体识别。该项目发表在ICLR 2024。

## 环境要求

### Python 依赖包
根据代码分析，需要安装以下主要依赖：
- `torch` (PyTorch)
- `numpy`
- `pandas`
- `einops` (用于张量重排)
- `transformers` (如果使用CLIP特征提取)
- `PIL` (Pillow，用于图像处理)
- `torch_geometric` (如果使用图注意力模块)

### 硬件要求
- GPU支持（代码中配置了CUDA）
- 足够的存储空间用于数据集和特征文件

## 数据准备

需要下载 **Things-EEG2** 数据集：
- 数据集链接：https://www.sciencedirect.com/science/article/pii/S1053811922008758

## 运行步骤

### 步骤1: EEG数据预处理

**脚本位置**: `./preprocessing/preprocessing.py`

**操作步骤**:
1. 根据需求修改 `preprocessing_utils.py`:
   - 选择通道
   - 设置epoching参数
   - 基线校正
   - 重采样到250 Hz
   - 按条件排序
   - 多变量噪声归一化(MVNN)

2. 运行预处理脚本（对每个被试分别运行）:
```bash
cd preprocessing
python preprocessing.py --sub 1 --n_ses 4 --sfreq 250 --mvnn_dim epochs --project_dir /path/to/Data/Things-EEG2/
```

**参数说明**:
- `--sub`: 被试编号 (1-10)
- `--n_ses`: 会话数量 (默认4)
- `--sfreq`: 采样频率 (默认250 Hz)
- `--mvnn_dim`: MVNN维度 ('time' 或 'epochs')
- `--project_dir`: 项目数据目录路径

**输出**: 预处理后的数据保存在 `Preprocessed_data_250Hz/sub-XX/` 目录下

### 步骤2: 图像特征提取（可选）

**注意**: 项目已经提供了CLIP特征文件在 `./dnn_feature/` 目录下：
- `clip_feature_maps_training.npy`
- `clip_feature_maps_test.npy`
- `center_clip.npy`

如果需要自己提取特征：

**脚本位置**: `./dnn_feature_extraction/`

**操作步骤**:
1. 提取每张图像的特征图:
```bash
python obtain_feature_maps_clip.py --pretrained True --project_dir /path/to/Data/Things-EEG2/
```

2. 将所有特征图打包成一个.npy文件:
```bash
python feature_maps_clip.py
```

3. 提取中心图像的特征:
```bash
python center_fea_clip.py
```

**输出**:
- `center_all_image_clip.npy`: 每张中心图像的特征
- `center_clip.npy`: 每个条件的特征（用于训练）

### 步骤3: 配置训练参数

**主训练脚本**: `nice_stand.py`

**需要修改的配置**（在脚本中）:
1. **GPU设置** (第26-28行):
```python
gpus = [0]  # 修改为你的GPU编号
os.environ['CUDA_DEVICE_ORDER'] = 'PCI_BUS_ID'
os.environ["CUDA_VISIBLE_DEVICES"] = ','.join(map(str, gpus))
```

2. **结果保存路径** (第29行):
```python
result_path = './results/'  # 修改为你的结果保存路径
```

3. **数据路径** (第163-165行):
```python
self.eeg_data_path = '/path/to/Preprocessed_data_250Hz/'  # EEG数据路径
self.img_data_path = './dnn_feature/'  # 图像特征路径
self.test_center_path = './dnn_feature/'  # 测试中心特征路径
```

4. **模型保存路径** (第340-342行):
```python
torch.save(..., './model/' + model_idx + 'Enc_eeg_cls.pth')
torch.save(..., './model/' + model_idx + 'Proj_eeg_cls.pth')
torch.save(..., './model/' + model_idx + 'Proj_img_cls.pth')
```
确保 `./model/` 目录存在。

### 步骤4: 运行训练和测试

**基本命令**:
```bash
python nice_stand.py --dnn clip --epoch 200 --num_sub 10 --batch-size 1000 --seed 2023
```

**参数说明**:
- `--dnn`: 使用的DNN模型 ('clip', 'vit', 'resnet')
- `--epoch`: 训练轮数 (默认200)
- `--num_sub`: 使用的被试数量 (默认10)
- `--batch-size`: 批次大小 (默认1000)
- `--seed`: 随机种子 (默认2023)

**训练过程**:
1. 模型会自动加载训练和测试数据
2. 使用对比学习进行训练
3. 每个epoch后会在验证集上评估
4. 保存最佳模型
5. 训练结束后在测试集上评估，输出Top-1, Top-3, Top-5准确率

**输出结果**:
- 模型权重保存在 `./model/` 目录
- 训练日志保存在 `result_path` 指定的目录
- 最终结果保存在 `result_path/result.csv`


## 常见问题

1. **路径问题**: 
   - 代码中有硬编码的Linux路径（如 `/home/Data/`），需要根据实际情况修改为Windows路径格式

2. **GPU内存不足**:
   - 减小 `--batch-size` 参数
   - 减少 `--num_sub` 参数，逐个被试训练

3. **缺少依赖包**:
   - 安装缺失的包: `pip install torch numpy pandas einops transformers pillow`

4. **数据格式问题**:
   - 确保预处理后的数据格式为: `(conditions, repetitions, channels, time_points)`
